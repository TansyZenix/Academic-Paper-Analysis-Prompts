# Academic Paper Analysis Prompts

> 面向计算机视觉 / AI 论文的**结构化分析提示词库**
>
> A structured prompt library for in-depth CV/AI paper analysis

---

## 简介

本项目提供一套完整的、结构化的 Prompt 提示词模板，用于系统性地分析和理解计算机视觉 / AI 领域学术论文。适用于科研入门者、论文复现者以及对论文深读有需求的研究人员。

每个模块针对论文的不同部分，设计了精细的指令约束，确保输出结果具备**可复现性**和**完整性**。

## 功能模块

| 模块 | 功能 | 适用论文部分 |
|------|------|-------------|
| 1. 文献筛选与推荐 | 按 4090 可复现性筛选高价值论文 | 文献调研 |
| 2. 领域理解与导读 | 初学者友好的论文核心解读 | 整体理解 |
| 3. 引言深入解析 | 从背景到创新的逻辑链分析 | Introduction |
| 4. 相关工作结构化解析 | 逐个方法带 shape 数据流拆解 | Related Work |
| 5. 任务形式化解析 | 输入→中间过程→输出的完整推导 | Task Formulation |
| 6. 模型框架深度解析 | 模块级可复现解析 + 实现提示 | Framework/Method |
| 7. 评价指标 | 指标公式 + 数值计算示例 | Experiments |
| 8. 实验 | 超参数、设置、可复现性分析 | Experiments |
| 9. 数据集 | 数据内容、预处理、划分方式 | Datasets |
| 10. 损失函数 | 总损失→子损失→数值示例逐层拆解 | Loss Function |

### 模块特点

- **统一的 Shape 规范**：全程使用 $\mathbb{R}^{B \times H \times W \times C}$ 统一表示
- **强制完整性检查**：每个模块都有防遗漏机制
- **数值示例**：关键模块包含可手动计算的最小化示例
- **复现导向**：明确标注"合理推测"与"未提及"
- **可复现级解析**：内容清晰到可直接画结构图或写 PyTorch 代码

## 使用方式

1. 将论文 PDF / Markdown 内容提供给 AI 助手
2. 选择对应论文部分的 Prompt 模块
3. AI 将按照模块中定义的严格格式输出结构化分析

```
例如：
- 想理解模型结构 → 使用模块 6（模型框架深度解析）
- 想复现实验 → 使用模块 8（实验）+ 模块 10（损失函数）
- 筛选可复现论文 → 使用模块 1（文献筛选与推荐）
```

## 依赖

- 支持 Prompt 的 AI 助手（如 Claude、GPT 等）
- 用于渲染数学公式的 Markdown 渲染器（推荐 VSCode + Markdown Preview Enhanced）

## 1. 文献筛选与推荐

```markdown
> 请分析提供的 Markdown 文档中的论文列表，并根据以下标准进行筛选与推荐，目标是选择**可以在单张 RTX 4090 上复现（训练或推理）的高价值论文**。
>
> ---
>
> ### **(1) 领域约束（必须满足）**
> * 仅保留以下方向论文：
>   * 计算机视觉（Computer Vision）
>   * 高影响力 AI/ML 会议论文（如 CVPR, ICCV, ECCV, NeurIPS, ICML, ICLR, IJCAI, ACL 等）
>
> ---
>
> ### **(2) 4090 可复现性（核心筛选标准）**
> 优先选择满足以下至少一项的论文：
> * 模型结构**轻量或中等规模**（非超大模型，如不依赖百亿参数）
> * 支持**单卡训练或推理**（无需多机多卡）
> * 论文或开源实现中提到：
>   * 可在 $\le 24\text{GB}$ 显存运行（RTX 4090）
>   * 或可通过常见策略运行（如 patch-based、downsample、gradient checkpoint）
> * 数据集规模**可控**（如 COCO 子集、ScanNet、KITTI 等，而非必须超大私有数据）
>
> ⚠️ **排除条件**：
> * 明显依赖大规模分布式训练（如数十/数百 GPU）
> * 强依赖超大预训练模型（如 GPT-级别、超大 diffusion）且无法裁剪
>
> ---
>
> ### **(3) 价值排序（Top 10）**
> 按以下优先级综合排序：
> 1. 是否被顶会接收（优先 oral / spotlight / accept）
> 2. 方法创新性与代表性
> 3. 对当前研究（CV / 3D / 多模态等）的参考价值
> 4. 是否已有代码或易于复现
>
> 最终输出 **Top 10 论文**
>
> ---
>
> ## **输出格式（严格遵守）**
> 对每篇论文，必须包含：
>
> ```markdown
> [Title]
> 作者: [Authors]
> 
> 评论: [Comments]  （如 accepted / oral / under review 等）
> 
> 文章链接: [Paper Link]
> 
> PDF链接: [PDF Link]
> 
> 推荐理由:
> - 研究方向/任务（属于哪个CV子领域）
> - 核心方法或创新点（简要）
> - 为什么具有参考价值（科研角度）
> - 为什么可以在 RTX 4090 上复现（必须具体说明：如模型规模 / 显存需求 / 可裁剪策略）
> ```
>
> ---
>
> ## **额外要求（提高质量）**
> * ⚠️ 不要给出空泛推荐理由（如"很有价值"），必须具体到方法或结构
> * ⚠️ 明确说明"是否训练可行"还是"仅推理可行"
> * ⚠️ 若可复现性基于推测，需标注"合理推测"
> * ⚠️ 优先推荐**有开源实现**的论文（如已知）
>
> ---
>
> ## **输出目标**
> 帮助用户筛选出：
> * 能在 RTX 4090 上**真正跑起来**的论文
> * 同时具备**科研参考价值（适合复现/改进/写论文）**
> * 优先推荐适合作为"复现或改进 baseline"的论文。
```

## 2. 领域理解与导读

```markdown
请像给初学者讲解一样说明（中文回答）：

1. 这篇论文关注的核心任务是什么？
2. 它试图解决的关键问题是什么？（为什么这个问题重要？）
3. 现有方法大致是怎么做的，有什么不足？
4. 这篇论文的基本思路是什么？（不用太细节）
5. 用一个简单类比或例子帮助理解。

*注意：请避免使用过多专业术语，如果必须使用，请进行解释。*
```

## 3. 引言深入解析

```markdown
请基于我上传的文献，对 Introduction（引言）部分进行深入解析（中文回答），要求：

1. 从研究背景、问题动机、技术挑战三个角度展开。
2. 分析该领域已有方法（如传统方法或相关工作）的不足。
3. 提炼论文的核心创新点，并说明其相对于已有工作的改进。
4. 结合计算机视觉领域背景，对关键概念进行解释。
5. 最后用一段话总结该论文引言的整体逻辑链条（problem $\to$ gap $\to$ solution）。

*注意：请避免使用过多专业术语，如果必须使用，请进行解释。*
```

## 4. 相关工作结构化解析

```markdown
请基于论文 Related Work 部分，对文中提到的每一个方法（或每一类代表性工作）分别进行结构化解析。

对每个方法，必须严格按照以下统一格式输出：

### **(1) 方法概述**
* 解决的问题
* 核心思想（1–2句话即可）

### **(2) 输入与输出**
* 输入数据类型 + shape（如 $\mathbb{R}^{B \times C \times H \times W}$）
* 输出结果 + shape

### **(3) 处理流程（带 shape）**
* 按步骤列出主要模块（模块级即可，不需要过细）
* 每一步都需标明输入与输出 shape

### **(4) 完整数据流示例（必须包含）**
* 给出一个从输入 $\to$ 中间过程 $\to$ 输出的完整示例
* 明确写出每一步的 shape 变化
* 示例需与该方法匹配，**不得与其他方法复用**

---

### **强制约束（必须遵守）**
* ⚠️ 不得遗漏任何一个 Related Work 中提到的方法
* ⚠️ 每个方法都必须包含"完整示例"，否则视为不完整
* ⚠️ shape 可合理假设（如 $\mathbb{R}^{1 \times 3 \times 224 \times 224}$），但必须前后一致
* ⚠️ 若论文未提供具体结构或 shape，请标注"合理推测"，不得编造细节
* ⚠️ 全程标注变量名称，特征维度 Shape 统一采用 $\mathbb{R}^{B \times H \times W \times C}$

### **输出目标**
内容应清晰到可以据此**绘制结构图或实现代码**。
```

## 5. 任务形式化解析

```markdown
请基于论文的 Task Formulation 部分进行结构化解析，要求按照"输入 $\to$ 中间过程 $\to$ 输出"的逻辑进行讲解，并满足以下规范：

### **(1) 输入与输出定义**
* 明确列出所有输入变量及其含义
* 给出每个变量的 shape，统一使用 $\mathbb{R}^{B \times H \times W \times C}$ 表示
* 明确最终输出及其 shape

### **(2) 中间过程（逐步展开）**
* 按公式或逻辑顺序，逐步说明从输入到输出的推导过程
* 每一步必须包含：
  * 使用到的变量名称
  * 对应的物理/数学含义（用通俗语言解释）
  * shape 变化（必须保持统一表示规范）

### **(3) 公式逐条讲解（必须单独列出）**
对每一个公式，必须包含：
* 公式中每个符号/变量的含义
* 公式在整体方法中的作用（解决什么问题）
* 输入与输出的 shape
* 该公式可以理解为在"做什么操作"（用直观语言解释，而不是只翻译）

### **(4) 完整数据流示例（必须包含）**
* 构造一个具体示例（可合理假设输入，如 $\mathbb{R}^{1 \times 224 \times 224 \times 3}$）
* 按步骤展示：输入 $\to$ 中间计算 $\to$ 输出
* 每一步都要：
  * 标注变量名称
  * 给出 shape
  * 对应到前面讲解的公式或步骤（保持一致）

---

### **强制约束（必须遵守）**
* ⚠️ 所有变量必须**前后一致命名**，不得更改符号
* ⚠️ shape 表达必须全程统一为 $\mathbb{R}^{B \times H \times W \times C}$
* ⚠️ 若论文未明确给出 shape，请标注"合理推测"，不得随意编造
* ⚠️ 不要跳步骤，保证推导过程完整可追踪
* ⚠️ 避免使用过多专业术语，若使用必须附带解释

### **输出目标**
内容应清晰到可以据此：
* 直接画出计算流程图
* 或实现对应的代码（如 PyTorch）
```

## 6. 模型框架深度解析

```markdown
> 请对论文的 **Framework（方法框架）部分进行完整、结构化、可复现级解析**，要求结合 **Preliminaries + 框架图 + 公式 + 模块流程** 进行统一讲解。目标是让我能够**彻底理解模型并复现实现**。
>
> ---
>
> # **(0) Preliminaries（必须首先给出）**
>
> 在进入 Framework 之前，必须先说明论文的基础设定：
>
> ### **(0.1) 任务定义**
>
> * 论文解决的问题是什么（如检测 / 分割 / SLAM / NeRF 等）
> * 输入与输出的数学形式
>
> ### **(0.2) 符号系统（必须统一）**
>
> * 所有核心变量与符号说明（如 $I, X, F, \theta$ 等）
> * 明确每个变量的：
>
>   * 含义
>   * shape（统一用 $\mathbb{R}^{B \times H \times W \times C}$ 或合理形式）
>
> ### **(0.3) 关键先验知识**
>
> * 涉及的基础方法（如 Transformer / CNN / 3DGS / 图优化等）
> * 用通俗语言解释其核心思想
>
> ---
>
> # **(1) 整体框架概述**
>
> * 模型由哪些模块组成（结构分解）
> * 结合框架图说明完整数据流：
>
> $$
> \text{Input} \rightarrow \text{Module 1} \rightarrow \text{Module 2} \rightarrow \cdots \rightarrow \text{Output}
> $$
>
> * 指出：
>
>   * 核心路径（main pipeline）
>   * 信息如何逐步变换（表示学习过程）
>
> ---
>
> # **(2) 子模块逐一解析（必须全部覆盖）**
>
> ⚠️ 每个模块必须严格按照以下结构：
>
> ---
>
> ## **[模块名称]**
>
> ### **① 输入**
>
> * 输入来源（来自哪个模块/数据）
> * 变量名 + shape
>
> ---
>
> ### **② 中间流程（必须"公式嵌入式讲解"）**
>
> 按步骤展开，每一步必须包含：
>
> #### **Step k：操作名称**
>
> * 在做什么（通俗解释）
> * 数学表达（若有）：
>
> $$
> \text{写出公式}
> $$
>
> * ⚠️ 紧接公式解释（必须）：
>
>   * 每个符号含义
>   * shape变化
>   * 该计算的作用（解决什么问题）
> * 输出特征 shape
>
> 👉 **严禁只列公式不解释，必须嵌入流程中**
>
> ---
>
> ### **③ 输出**
>
> * 输出变量
> * shape
> * 流向哪个模块（必须明确）
>
> ---
>
> # **(3) 图表逐一解析（必须覆盖所有图/表）**
>
> 对论文中每一个图 / 表：
>
> ### **[图/表编号，如 Fig.1]**
>
> * 图中展示什么内容
> * 每个组件代表什么模块
> * 数据流如何流动（对应 Framework）
> * 图与公式/模块的对应关系
>
> ---
>
> # **(4) 公式逐条独立解析（防遗漏机制）**
>
> ⚠️ 即使在模块中解释过，这里仍需完整列出，防止遗漏
>
> 对每一个关键公式：
>
> ### **[公式编号]**
>
> * 公式：
>
> $$
> ...
> $$
>
> * 变量说明（逐个）
> * 输入 / 输出 shape
> * 作用（解决什么问题）
> * 通俗解释（本质在做什么计算）
>
> ---
>
> # **(5) 全流程串联（系统级理解）**
>
> 按完整 pipeline 串联：
>
> $$
> \text{Input} \rightarrow M_1 \rightarrow M_2 \rightarrow \cdots \rightarrow \text{Output}
> $$
>
> 必须说明：
>
> * 数据表示如何逐步变化
> * 信息如何：
>
>   * 提取
>   * 融合
>   * 优化
> * 哪些模块是关键（性能瓶颈 / 创新点）
>
> ---
>
> # **(6) 实现提示（用于复现，新增强化）**
>
> * 每个模块可对应的 PyTorch 操作（如 Conv / Attention / MLP）
> * 是否涉及：
>
>   * reshape / flatten / projection
>   * loss 计算
> * 关键实现难点
>
> ---
>
> # **强制约束（必须遵守）**
>
> * ⚠️ 必须包含 Preliminaries
> * ⚠️ 不得遗漏任何模块 / 图 / 公式
> * ⚠️ 每个模块必须包含：输入 + 流程 + 输出
> * ⚠️ 所有公式必须解释（且嵌入流程）
> * ⚠️ shape 必须统一规范
> * ⚠️ 未给 shape 必须标注"合理推测"
> * ⚠️ 模块连接关系必须明确
> * ⚠️ 避免术语堆砌，必须解释清楚
>
> ---
>
> # **输出目标**
>
> 解析结果必须达到：
>
> ✅ 可以画出完整结构图
> ✅ 可以写出 PyTorch 代码
> ✅ 可以理解每一个公式在"干什么"
> ✅ 中文回答
```


## 7. 评价指标

```markdown
> 请对论文中使用的**所有评价指标（Evaluation Metrics）**进行完整、结构化讲解，要求不遗漏任何一个指标。
>
> ---
>
> ## **对每一个评价指标，必须按照以下结构说明：**
>

>
> * 指标名称
> * 该指标用于衡量什么（解决什么评估问题）
>

>

> ### **(5) 数值计算示例（必须包含）**
>
> * 构造一个**具体的小例子**（包含真实值与预测值）

>
> ---
>
> ## **强制约束（必须遵守）**
>
> * ⚠️ 不得遗漏论文中出现的任何一个评价指标
> * ⚠️ 每个指标都必须包含"公式 + 示例"，否则视为不完整
> * ⚠️ 示例必须**独立构造**，不能多个指标共用一个例子（除非合理且说明）
> * ⚠️ 若论文未明确给出公式，可补充标准定义并标注"通用定义"
> * ⚠️ 避免只给公式不解释，必须结合直观理解说明
>
> ---
>
> ## **输出目标**
>
> 内容应清晰到可以：
>
> * 手动复现指标计算过程
> * 或直接用于实现评估代码
> * 中文回答

```

## 8. 实验

```markdown
> 请对论文的 **Experiments（实验）部分**进行系统化解析，目标是帮助理解实验设置并支持复现。请按照以下结构输出：
>
> ---
>
> ## **(1) 超参数汇总与解释（必须完整列出）**
>
